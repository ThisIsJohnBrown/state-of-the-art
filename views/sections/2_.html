Modular rendering system


100% JS

started out as a single canvas -> projection mapping

Image capture/GIF encoding


Tried a lot of things

Capturing every frame - too slow!

WebWorkers didn't make much of a difference

Ended up capturing at a low framerate, then sending image data up one by one with progress bar

server runs on node.js

Express and socket.io handle communication to the front end, and serial peg data from the board

GIF encoding is done in a child process (process.fork)
